# flt v0.3 — Presets + TUI Phases 6-9

## Feature 1: Agent Presets

### Config
`~/.flt/presets.json`:
```json
{
  "coder": {
    "cli": "codex",
    "model": "gpt-5.3-codex",
    "description": "Fast coder, full bypass"
  },
  "reviewer": {
    "cli": "claude-code",
    "model": "opus",
    "description": "Thorough code review"
  },
  "scout": {
    "cli": "claude-code",
    "model": "haiku",
    "description": "Cheap research"
  }
}
```

### CLI Commands
```
flt presets list                              # list all presets
flt presets add <name> --cli <cli> --model <model> [--description <desc>]
flt presets remove <name>
```

### TUI Commands
```
:presets list                                 # show presets in log pane
:presets add <name> --cli <cli> --model <model> --description <desc>
:presets remove <name>
```

### Spawn Integration
```
:spawn myagent --preset coder ~/project "fix the bug"
flt spawn myagent --preset coder --dir ~/project "fix the bug"
```
`--preset` fills in `--cli` and `--model`. Can be overridden:
```
:spawn myagent --preset coder --model gpt-5.4-mini ~/project "task"
```

### Tab Completion
- `:spawn name --preset ` Tab → completes preset names
- `:presets remove ` Tab → completes preset names

### Implementation
- `src/presets.ts` — load/save/add/remove/list presets from `~/.flt/presets.json`
- `src/commands/presets.ts` — CLI command handler
- Update `src/cli.ts` — register `flt presets` subcommand
- Update `src/commands/spawn.ts` — resolve `--preset` flag
- Update `src/tui/app.ts` — handle `:presets` and `:spawn --preset` commands
- Update `src/tui/input.ts` — tab completion for preset names
- Tests: `tests/unit/presets.test.ts`
